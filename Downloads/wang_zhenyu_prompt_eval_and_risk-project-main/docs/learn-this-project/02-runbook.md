## First-time setup

```bash
# 1. Clone the repo
git clone <repo-url>
cd wang_zhenyu_prompt_eval_and_risk-project

# 2. Install tools (Python 3.12, uv) declared in mise.toml
mise install

# 3. Create the virtual environment
mise run venv-create

# 4. Install all Python dependencies (including dev, test, docs extras)
mise run inst

# 5. Verify: run unit tests (no AWS needed)
mise run test
```

## Running locally

### Unit tests (offline, fast)
```bash
mise run test
# or with coverage:
mise run cov
```

### A single example script (requires AWS)
```bash
# Activate the venv first (mise should do this automatically; if not:)
source .venv/bin/activate

# Run a P1 extraction example (edit the file to pick a test case)
python examples/uc1-claim-intake/run_uc1_p1_extraction.py

# Run J1 judge on a UC1-P1 prompt version
python examples/judges/run_j1_on_uc1_p1.py
```

To switch which test case runs: open the file, comment/uncomment enum entries near the top:
```python
case = P1LoaderEnum.B_01_AUTO_REAR_END
# case = P1LoaderEnum.A_01_INJECTION_IN_NARRATIVE
```

To switch prompt version in the J1 demo:
```python
prompt_version = "01"  # production-quality
# prompt_version = "02"  # over-permissive (CRITICAL)
# prompt_version = "03"  # minimal (MEDIUM)
# prompt_version = "04"  # instruction conflict
```

### Interactive notebooks (requires AWS)
```bash
mise run lab
# Open docs/source/06-Prompt-Runner-And-Evaluation-Demo/index.ipynb
# or docs/source/08-Judge-Demo/index.ipynb
```

## Tests

**Location**: `tests/`

**Modules**:
- `tests/test_evaluations.py` — 25+ tests for `evaluate()` (eq/in/ne assertions, edge cases)
- `tests/test_llm_output.py` — 13+ tests for `extract_json()` (JSON fences, bare text, error cases)
- `tests/test_prompts.py` — loads and renders two prompt templates (reads from `data/`)
- `tests/test_api.py` — smoke test importing the `one` API singleton

**Run all**:
```bash
mise run test
```

**Run one module with its own coverage**:
```bash
python tests/test_evaluations.py
```
Each test module uses `run_cov_test()` from `prompt_risk/tests/helper.py` when run as `__main__`, generating a per-module coverage report.

**What is NOT tested**: live Bedrock API calls, end-to-end pipeline runs, prompt rendering with real data at scale. No integration tests yet (`tests_int/` is empty).

## Common failures and fixes

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| `NoCredentialsError` or `ProfileNotFound` | AWS profile `wang_zhenyu_dev` missing | `aws configure --profile wang_zhenyu_dev` |
| `ModuleNotFoundError: prompt_risk` | Package not installed in active venv | `mise run inst` then confirm `.venv` is active |
| `FileNotFoundError` on a `.jinja` or `.toml` file | Wrong working directory when running a script | Run from repo root; or confirm `path_enum.dir_project_root` resolves correctly |
| `JsonExtractionError` | LLM returned non-JSON text | Check the raw output in the exception; try increasing `MAX_RETRIES` or improving the system prompt |
| Pydantic `ValidationError` after 3 retries | LLM repeatedly fails to match the output schema | Inspect the `field_errors` in the exception; check the system prompt's output specification matches the Pydantic model |
| Sphinx build fails on notebook | `nbsphinx` / `jupyter` not installed | `mise run inst` (all extras) |

## Where to look when something breaks

| Problem area | Where to look |
|-------------|---------------|
| LLM response not parsing | `prompt_risk/llm_output.py` — add a breakpoint before `extract_json` returns |
| Wrong field in output | The system prompt's JSON schema section in `data/.../system-prompt.jinja` |
| Retry loop not triggering | `prompt_risk/uc/uc1/p1_extraction_runner.py:MAX_RETRIES` |
| AWS auth | `prompt_risk/one/one_03_boto_ses.py` — profile name is hardcoded here |
| Test case data | `data/{use_case}/prompts/{prompt}/normal/` and `attack/` |
| Coverage report | `htmlcov/index.html` (open after `mise run cov`) |
| Sphinx build errors | `docs/build/html/` — or check terminal output from `mise run build-doc` |
